# Тестови сценарии — Етап 6 Мачове

## Архитектура

```
handler → service → repo
```

- `handler` — валидира вход и контекст
- `service` — бизнес логика, всички проверки
- `repo` — SQL заявки (без логика)

Контекстът се съхранява в `Router`:
- `current_league_name` / `current_season`
- `current_match_id`

## Контекстен подход (препоръчителен)

1. Потребителят избира лига: `Избери лига <име> <сезон>`
2. След това командите `Покажи кръг <N>`, `Покажи събития` работят без да указват лига/сезон
3. Избор на мач: `Избери мач <ID>` → след това `Резултат <X>:<Y> запиши`, `Гол ...`, `Картон ...`

## Категории валидации

### Резултат
- `X` и `Y` трябва да са цели числа ≥ 0
- Мачът да не е вече `played`
- Ако мачът съществува с обърнат ред на отборите, се указва правилният ред

### Гол
- Минута: 1–120
- Играч да съществува
- Играч да е от посочения отбор
- Отборът да участва в избрания мач

### Картон
- Тип: Y (жълт) или R (червен)
- Минута: 1–120
- Играч да съществува
- Играч да е от посочения отбор
- Отборът да участва в избрания мач

## Режим на резултата

Използва се **Режим 1 — „Резултатът е главен“**:
- Резултатът се записва ръчно (home_goals:away_goals)
- Головете и картоните са статистика
- Резултатът НЕ се преизчислява от головете

---

## 1. Покажи кръг → списък мачове (OK)

```
Покажи кръг 3 Първа лига 2025/2026
```

Очакван резултат: списък с match_id, домакин, гост, статус, резултат.

---

## 2. Избери мач (OK)

```
Избери мач 15
```

Очакван резултат: `Избран мач #15: Ботев vs ЦСКА (? : ?) — кръг 2, scheduled`

---

## 3. Запиши резултат (OK)

След избор на мач:

```
Резултат 3:0 запиши
```

Очакван резултат: `Записано: Ботев–ЦСКА 3:0 (мач #15)`

---

## 4. Запиши повторно резултат (ERROR)

```
Резултат 1:0 запиши
```

Очакван резултат: `... вече е изигран с резултат 3:0.`

---

## 5. Грешен формат за резултат (ERROR)

```
Резултат -1:5 запиши
```

Очакван резултат: `Резултатът не може да бъде отрицателен.`

---

## 6. Гол с играч от друг отбор (ERROR)

```
Гол Христо Стоичков Ботев 23 минута
```
(Христо Стоичков е в ЦСКА, опитваме гол за Ботев)

Очакван резултат: `Играч ... не играе за ...`

---

## 7. Гол с минута 0 или 200 (ERROR)

```
Гол <играч> <отбор> 0 минута
```

Очакван резултат: `Минутата трябва да е между 1 и 120, получена: 0.`

---

## 8. Гол валиден (OK)

```
Гол Стилиян Петров Левски 23 минута
```

Очакван резултат: `Гол: Стилиян Петров (Левски) 23' — мач #...`

---

## 9. Картон Y валиден (OK)

```
Картон Стилиян Петров Левски Y 41
```

Очакван резултат: `Картон: Стилиян Петров (Левски) 41' — жълт (мач #...)`

---

## 10. Покажи събития (OK)

```
Покажи събития
```

Очакван резултат: хронологичен списък:
```
Събития за мач #...: Ботев vs ЦСКА (3:0)
  Голове:
    23' Стилиян Петров (Левски)
    55' Христо Стоичков (ЦСКА)
  Картони:
    41' ... — Y
    78' ... — R
```

---

## 11. Резултат по клубове (обратен ред — ERROR)

```
Резултат Левски-Ботев 1:1 запиши
```
(реалният мач е Ботев vs Левски)

Очакван резултат: `Мачът между тези отбори е Ботев vs Левски. Използвайте правилния ред: домакин-гост.`

---

## 12. Резултат по клубове — несъществуващ отбор (ERROR)

```
Резултат Левски-Литекс 1:1 запиши
```

Очакван резултат: `Няма мач 'Левски vs Литекс' в ...`

---

## 13. Без контекст (ERROR)

Нов сесия (без избран мач):

```
Резултат 1:0 запиши
```

Очакван резултат: `Няма избран мач. Първо използвайте 'Избери мач <ID>'.`
